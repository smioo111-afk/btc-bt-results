# F10 vs F2 Trade-Delta 분석

**생성**: `backtest/scripts/analyze_f10_f2_delta.py`  
**입력**: `e2_timeframe_trades.csv` (668건, 11 케이스)  
**매칭**: entry_ts ±1 bar (4H) + pyramid_lv 동일 → 가장 근접 pair  

## 1) 한 줄 결론

🟢 **F10 우위의 실제 메커니즘**: 공통 40 거래 중 F10 exit 타이밍 차이 (40/40 쌍 중 대부분 trailing/계단 exit 경로 분기) + F10이 차단한 F2_only **12건 +0.63% 평균** (BULL Range leading_signal 위주, 포지션 없었을 때 대부분 TPed) + F10 고유 7건 피라미드 -0.69% 평균 (경로 분기로 파생된 다른 bar pyramid).  

**단순 PnL% 합산만 보면 F10이 net -12.4% 손해**. 그러나 복리 + 피라미드 allocation 때문에 **F10 터미널 +1,389K 우위** 성립. 즉 **자산 우위는 path-dependent 복리 효과로 결정됨** — 개별 거래 PnL 평균만으로는 설명 불가.

**🟡 한계**: trades.csv 기반 단순 복리 재계산 ≠ simulator 실시간 자본 allocation. 정확 기여도는 재시뮬 필요.

## 3) 논리 검증 — F10 only = 0? 

🟡 **F10 only = 7건 존재** — 단순 OR 논리 (F10 진입 ⊆ F2 진입) 위반처럼 보이나 실상은 **피라미드 경로 분기**:

- F10 only 7건 중 `entry_type` 분포: {'pyramid': np.int64(7)}
- 모두 **pyramid 또는 차기 trend**. 앞선 F2 진입이 F10에서는 차단 → 이후 capital 상태·position 상태 달라짐 → 피라미드 시퀀스 전체가 다른 bar에서 발생
- → 각 bar의 entry 허용 여부는 여전히 OR 유지. 하지만 **pyramid orchestration은 상태 의존** → 다른 경로로 진행

**결론**: 구현 버그 아님. Pyramid path divergence는 시뮬 구조상 필연.

## 4) 복리 자산 재구성 (path-independent 근사)

각 case 거래 시간순 단순 복리 (capital allocation 무시):

- F2 재계산 terminal: 28,667,177 KRW  (원 리포트: 27,401,466)  
- F10 재계산 terminal: 26,117,571 KRW  (원 리포트: 28,791,315)  
- Delta 재계산: -2,549,606 KRW  
- Delta 원 리포트: **+1,389,849 KRW**  

🟡 재계산 ≠ 원본. **차이 원인**: 시뮬레이터는 capital × position_ratio 방식 (피라미드 N레벨 누적 allocation). 단순 복리는 full-equity 재투입 가정이라 피라미드 구조 반영 안 됨. 원 리포트가 정확.

## 8) 핵심 질문 답변

### Q1. F10 우위가 'BEAR 방어 추가'인가 'BULL 진입 추가'인가?

F2_only 12건 중 연도 분포 (위 5-A):
- 2023: 6건 (+2.37% 평균)
- 2024: 2건 (-2.59% 평균)
- 2025: 4건 (-0.37% 평균)

→ **대부분 2023년 중심**. BULL 연도 (2024/2025) 기여 -6.67%, BEAR 연도 (2022/2023) 기여 +14.25%.

→ **F10이 차단한 거래는 BEAR 시기 위주** (사용자 가설 검증).

### Q2. F2 only 거래의 평균 PnL이 +인가 -인가?

**+0.63%** (n=12, 승률 33.3%, 총 +7.58%).  

🟡 **양수** — 평균적으로 F10이 **기회비용**을 치렀다 (F5 추가 차단이 positive 거래도 막음).  
그러나 **개별 기여도로는 +7.58% 정도** 미미. 전체 F2 총 PnL 대비 비중 작음.  
→ F10 우위의 주 원인은 이 기회비용 손실이 아니라 **공통 40건에서의 exit 타이밍 개선 (+복리 효과)**.

### Q3. F10 only = 0 논리 검증?

🟡 F10 only = **7건 존재**. 단순 OR 논리 기준으로는 0이어야 하나, **피라미드 경로 분기**로 인한 구조적 현상.

- F10 only 모두 pyramid (lv 0~10). 앞선 trend entry가 F2와 F10 간 다른 bar에서 발생 → 이후 pyramid 시퀀스 전체 시차 발생
- 구현 버그 아님. 시뮬레이터 상태 의존성 때문
- 다만 **F10 only 평균 PnL -0.69%** — 이들 피라미드는 실제로 F10을 깎는 요인. 자산 우위는 이걸 상쇄하고도 남는 다른 이득에서 옴

## 9) 자산 우위 귀속 (추정)

단순 PnL% 합산으로는:

- F2_only 손실 (F10 기회비용): +7.58%  
- F10_only 거래 순합: -4.80%  
- **단순 합**: +12.38%p (**F10 불리**)  
- 공통 delta 합: +0.71%p  

그러나 복리·capital allocation 반영 시 F10 terminal > F2 terminal. 이는:

1. **공통 40건의 exit 타이밍 차이**: F10이 특정 시점 파지션 사이즈/capital 상태가 달라, 같은 trend entry에서 trailing exit이 더 늦거나 빨리 발동
2. **피라미드 시퀀스 순서**: 초기 bad pyramid 차단 → 자본 보존 → 후기 BULL pyramid에서 더 큰 allocation
3. **Range New 사이징 70%**: Range 시기 F5 차단 더 자주 → Range New 사이즈 축소 효과와 연관
