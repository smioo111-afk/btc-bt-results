## 1. per-position 파라미터 인프라 🟢

- `status["pos_trail_m"]` (TU 진입 시 4.5, 기타 0.0 → 전역 fallback)

- 사용 위치:
  - `_update_trailing_stop` L1710: `trail_m=self.status.get("pos_trail_m") or None`
  - `_check_partial_tp` L2009: `_step_lb = self.status.get("pos_step_lookback") or 0; _step_lb = _step_lb if _step_lb > 0 else STEP_LOOKBACK`

**v20.7 확장 방안**:
- **pos_step_lookback 재활용 (권장)**: Range 진입 시 3.0 설정, TU 진입 시 2.5 유지. 신규 필드 불필요.

**결론**: 기존 인프라 그대로 활용. pos_step_lookback 재활용이 단순. pos_init_ratio는 status에 기록용.

## 3. Range step_lookback 분리 🟢

**현행 (v20.6)**:
- 전역: `STEP_LOOKBACK = 1.5` (L285)
- `_check_partial_tp` L2009:
  ```python
  _step_lb = self.status.get("pos_step_lookback") or 0
  _step_lb = _step_lb if _step_lb > 0 else STEP_LOOKBACK
  ```
- TU 진입 시 `pos_step_lookback = TU_STEP_LOOKBACK = 2.5` 저장
- Range 진입 시 `pos_step_lookback = 0.0` → 전역 1.5 사용 (의도치 않은 상태)

**v20.7 변경**:
- 신규 상수: `RANGE_STEP_LOOKBACK = 3.0`
- 진입 블록 L3435 변경:
  ```python
  _pos_step_lb = (TU_STEP_LOOKBACK if _is_tu_pyramid
                  else RANGE_STEP_LOOKBACK if _is_range_new
                  else 0.0)
  ```
- L2009의 읽기 로직은 변경 없음 (pos_step_lookback 값에 따라 자동 분기)

**결론**: pos_step_lookback 재활용. 진입 블록 1줄 수정.
