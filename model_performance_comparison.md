## 핵심 질문 답변

### Q1. Main XGB가 충분히 좋은가?
**부족.** 전체 PF 1.46, Prec 50%는 baseline 수준. Range에서만 PF 2.10으로 양호하지만, Trend_Up (PF 0.02) 과 Volatile (PF 0.74)에서 손실. 신호 강도 0.55~0.70 구간이 노이즈.

### Q2. RF/Online이 더 좋다면 얼마나 차이?
- RF는 XGB 대비 **미세하게 우세** (Prec +1.8%p, PF +0.16) — 통계 유의성 약함.
- Online은 **명확하게 우세** (Prec +15%p, PF +1.13). 특히 Range에서 PF 5.64 vs 2.10. 단 Online의 신호 발사 빈도가 낮음 (40/89 vs XGB 50/89).

### Q3. 표본 충분한가?
- 전체 89: **1차 트렌드 확인 가능, 결정적 결론은 N≥200 권장**
- Range 61: 의사결정 가능 수준
- Trend_Up 16 / Volatile 8 / Trend_Down 4: **결론 불가**, 추가 누적 필요

### Q4. 통계 유의 차이?
- Online vs XGB Win Rate 65% vs 50% (n=89 기준):
  - 표본 비율차 검정 z ≈ 2.05, p ≈ 0.04 — **5% 유의수준에서 유의**
- RF vs XGB Win Rate 51.8% vs 50%:
  - z ≈ 0.24, p ≈ 0.81 — **유의차 없음** (= 운)
- Range 한정 Online vs XGB: 73.1% vs 57.9% (n=61):
  - z ≈ 1.81, p ≈ 0.07 — **경계선 (10% 유의)**

→ Online과 XGB의 차이는 통계적으로 의미 있고, RF와 XGB는 거의 동일.

### Q5. Main 개선 필요한가?
**예 — 단 신중하게.** 후보 3가지:

1. **신호 강도 임계 상향** (저비용/저위험)
   - 현재 0.55~0.60 구간이 노이즈 (Win 45%). 임계 0.60 또는 0.65로 상향 검토.
   - 단 신호 빈도 감소 → 백테스트 검증 필요.

2. **Regime별 모델 가중** (중비용/중위험)
   - Range에서 Online prob을 부분 가중 (예: ensemble = 0.6×XGB + 0.4×Online).
   - Trend_Up에서는 모든 모델 약함 — 진입 더 보수적으로.
   - 단 Online은 라이브 누적 의존 — 데이터 분포 변화 시 변동성 큼.

3. **Trend_Up 진입 차단/약화** (저비용/검증 필요)
   - 표본 16이지만 Win 14% (XGB) — 매우 약함.
   - E2 OFF 결정 (#74)과 결합 시 위험. 추가 누적 후 재평가.

## 결론 + Main 개선 권고

1. **즉시**: 추가 변경 없이 표본 누적 — 특히 Trend_Up / Volatile / Trend_Down. 200건 도달 시 재분석.
2. **단기 (1~2주)**: XGB 임계 0.60 또는 0.65 백테스트 검증 — 신호 빈도 vs PF 트레이드오프 측정.
3. **중기 (4~6주)**: Range 한정 Online 가중 ensemble 백테스트 — Online 라이브 PF 5.64를 검증.
4. **불권고**: XGB 자체 모델 교체. 표본 89로는 모델 교체를 정당화하기 부족하며, Online은 Recall 낮은 특성이 있어 단독 사용은 위험.

**핵심 메시지**: Main XGB는 baseline이며 분명한 약점(Trend_Up, 노이즈 구간)이 있다. 그러나 측정 기준 자체(holdout test PF vs 라이브 OOS PF)가 다르므로, 사용자가 본 Main PF=0.43은 학습 직후 holdout 결과이고 라이브 환경에서의 실제 성능은 PF 1.46 — 양적 차이 인지 필요.
