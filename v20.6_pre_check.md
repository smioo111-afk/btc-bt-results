### 루트 코즈

`_sync_balance()` (L1431-1436) 에 `_partial_selling` 가드 없음. `_detect_manual_trade()` (L1551)에는 있지만 `_sync_balance`에서 호출되는 팬텀 체크는 가드 없이 실행됨.

### 수정안 비교

| 안 | 변경 | 장점 | 단점 | 권고 |
|---|---|---|---|---|
| A | `_sync_balance` L1435에 `if not self._partial_selling` 가드 추가 | 최소 변경, 버그 원점 차단 | 다른 호출 경로(예: 수동매도 감지)에도 영향 확인 필요 | **✅ 1순위** |
| B | 하드스톱 fallback에서 `_sync_balance` 대신 `upbit_get_all_balances` 직접 호출 | 경로 명시적, phantom 완전 우회 | fallback 지점에만 localized, 다른 경로는 여전히 위험 | 2순위 (A 보완) |
| C | sell_type 판정 시 `entry<=0` 체크 후 `asp - entry_cached` 사용 | 방어적 | 본질적 원인(entry 조기 0화) 미해결, 다른 코드가 여전히 망가진 status 사용 | ❌ 증상만 봉합 |

**권고**: **A (주) + C (보조)**. 
- A: L1432-1436 에 `if self._partial_selling: return` 추가 (ideally after balances fetch but before phantom check).
- C: L1704-1709 방어 — entry가 0이면 fallback으로 `self._last_known_entry` 사용 (신규 필드).

### 관련 영향 — status.json 수동 정정

현재 status.json:
```
"last_sell_reason": "ATR트레일링(익절)"  ← 04-19 실제는 하드스톱 손절
"cooldown_seconds": 3600                ← STOPLOSS 14400 이어야 함
"consecutive_losses": 1                  ← 올바름 (raw 계산이 이미 net 음수로 반영)
```
이미 쿨다운은 만료됨 (18:52 + 1h = 19:52 < 현재). 하지만 `last_sell_reason`은 미래 로직이 참조할 수 있으므로 **v20.6 배포 시 동시에 수동으로 "인트라캔들하드스톱"으로 정정** 권고.

## 5. 시뮬레이터 vs 프로덕션 정합성 🔴

| 영역 | 상태 | 세부 |
|---|---|---|
| Step Mode | **DRIFTED** | 프로덕션: infinite(`STEP_LOOKBACK=1.5` 항상 사용) / 시뮬 V20_CFG 기본: `legacy` |
| 트레일링 공식 | CONSISTENT | `max(trail_high-ATR×TM, trail_high×(1-TRAIL_PCT), entry×(1-MIN_STOP))` 동일 |
| Range New 트레일 제외 | **DRIFTED (legacy)** | 프로덕션 L1640 ✅ / 시뮬 infinite P2 L492 ✅ / **시뮬 legacy P2 L556 ❌ Range 배제 없음** |

| 인트라캔들 하드스톱 | PROD-ONLY | 시뮬 close-bar 기반 (의도적 한계) |
| D1 Trend_Down 50% | SIM-ONLY | 시뮬 실험 기능 / 프로덕션 미구현 (OK) |
| 진입 로직 | CONSISTENT | EMA/leading/regime/score/RSI/BB/VWAP 일치 |

### TU 1/2차 백테스트 결과 재검증 필요 🔴

- 시뮬 step_mode=legacy 로 실행된 R3 B0, BD 결과는 프로덕션 infinite 모드 기준 실효성 **미확인**.
- v20.6 배포 전에 infinite 모드 B0 재검증 + BD 재검증 필요:
  ```
  cfg = {**V20_CFG, "range_new_enabled": True, "range_score_th": 5.0,
         "step_mode": "infinite", "step_lookback": 1.5,
         "tu_trail_m": 4.5, "tu_step_off": -1.0}  # TU_STEP_LOOKBACK 2.5 필요 구현
  ```
- 현재 시뮬 `tu_step_off` 는 legacy P2 전용. infinite P2에는 적용 없음. **TU-specific STEP_LOOKBACK 2.5 기능 시뮬에 추가 필요** (v20.6 구현 전제).
